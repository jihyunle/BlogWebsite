Git Hub 2
January 30, 2019

Git Hub is a wonderful resource. I always imagined that the process of collaborating on a coding project would be a giant pain. In some ways I was right (See Josiah's post about rebase) but Git Hub, for the most part, gives users ample opportunities to make sure no one team member's work goes unaccounted for.

Learning it on the first few days of the bootcamp gave me a deeper understanding about what all those seemingly random "git" commands I saw on Stack Overflow meant. My tip for new users would be - remember these three early on: Add, Commit, Push. Pull first if you are collaborating. And if possible, only add the src folder so you don't have to sit through mires of merge conflict! Also, if you find yourself working on another person's machine, be mindful of the configuration and whose GitHub account you're posting from.

Program Design
February 4, 2019

When writing programs you should first come up with a solid understanding of the problem you are aiming to solve. Then you should understand the data that will be used to solve that problem. Duriung the requirements and design phase you should:

Define the problem to be solved.
Outline the solution
Develop an algorithm outline.
Test the alogrithm.
Document the solution.

Then there is the implementation and verification phase:
Code the algorithm
Run the program
Test the program
Make corrections
Repeat

This is followed by the Maintenance phase in which the solution is documented and the program is maintained.

Program Flow
February 6, 2019

I like to write down the IPO - Input, Output, Processing for each project first. It helps if I can draw a flowchart (diagram) that show the hierarchy of each processes. It helps to have this visual aid throughout the project, because you can easily communicate to others what you're trying to accomplish.

One time I was working on a chat simulation program, and I had to work through several if-statements. I showed someone my flowchart and my code, and it became clear that the chart was right, but the implementation wasn't. It's a good practice that I think is important to get it down in the beginning stage of learnign to code.

Core Java 1
February 10, 2019

JAVA was developed by Sun Microsystems Inc in 1991, later acquired by Oracle Corporation. It was developed by James Gosling and Patrick Naughton. It is a simple programming language. Writing, compiling and debugging a program is easy in java. It helps to create modular programs and reusable code.

Writing of a program is done by a java programmer. Then the compilation of program is done by javac compiler, javac is the primary java compiler included in java development kit. It takes java program as input and generates java bytecode as output. In third phase, JVM executes the bytecode generated by compiler. This is called program run phase. Each operating system has different JVM, however the output they produce after execution of bytecode is same across all operating systems.

Object Orientated 1
February 15, 2019

Object-oriented programming (OOP) is a programming paradigm based on the concept of "objects", which may contain data, in the form of fields, often known as attributes; and code, in the form of procedures, often known as methods. A feature of objects is that an object's procedures can access and often modify the data fields of the object with which they are associated (objects have a notion of "this" or "self").

In OOP, computer programs are designed by making them out of objects that interact with one another. There is significant diversity of OOP languages, but the most popular ones are class-based, meaning that objects are instances of classes, which typically also determine their type.

Languages that support object-oriented programming typically use inheritance for code reuse and extensibility in the form of either classes or prototypes. Those that use classes support two main concepts:
Classes - Which contain the definitions for the data format and available procedures for a given type or class of object and may also contain data and procedure.
Objects: Instances of classes

Object Orientated 2
February 17, 2019

Object Oriented Programming, or OOP, gives you freedom that you can appreciate once you grasp the concepts of core java. Instead of playing with the primitive data types, you get to CREATE your own data types. Other beginners will appreciate this, but you could create a car class, a vehicle class, an engine class, a windows class, and much more. -- which makes sense, because the bigger the project, the bigger the need to compartmentalize the right way. Using OOP, you can do every nitty gritty thing you want to do in your Engine class, then simply extend the Car class to it. Like a parent who oversees a child's bank account, the superclass (car) can control the subclass (engine), and add other functions of its own.

HTML
February 19, 2019

Hypertext Markup Language (HTML) is the standard markup language for creating web pages and web applications. With Cascading Style Sheets (CSS) and JavaScript, it forms a triad of cornerstone technologies for the World Wide Web. Web browsers receive HTML documents from a web server or from local storage and render the documents into multimedia web pages. HTML describes the structure of a web page semantically and originally included cues for the appearance of the document.

HTML elements are the building blocks of HTML pages. With HTML constructs, images and other objects such as interactive forms may be embedded into the rendered page. HTML provides a means to create structured documents by denoting structural semantics for text such as headings, paragraphs, lists, links, quotes and other items. HTML elements are delineated by tags, written using angle brackets.
